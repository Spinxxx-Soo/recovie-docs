# 체크리스트 (Checklist)

본 문서는 **영화 시나리오 분석 플랫폼(Recovie)**의  
설계 → 구현 → 배포 전 단계에서 확인해야 할 체크리스트를 정의합니다.

이 문서는 다음 용도로 사용됩니다.
- 설계 문서 완성도 점검
- 개발 진행 상황 관리
- 코드 리뷰 / 과제 제출 / 포트폴리오 검증

---

## 1. 설계 문서 완성도 체크

### 1.1 아키텍처(architecture.md)
- [ ] 전체 시스템 구성(Frontend / API / DB / Cache / Worker / AI 추천) 설명됨
- [ ] Docker 기반 배포 구조 고려됨
- [ ] Mermaid 또는 다이어그램 이미지 포함됨
- [ ] 주요 데이터 흐름(조회/저장/추천) 설명됨
- [ ] 확장 로드맵(AI 추천 고도화) 명시됨

---

### 1.2 API 명세(api.md)
- [ ] Base URL 및 인증 방식(JWT) 명시
- [ ] 비로그인 가능 API와 로그인 필수 API 구분됨
- [ ] 영화 목록 조회 API
  - [ ] 검색 파라미터 정의
  - [ ] 페이징 처리 정의
- [ ] 영화 상세 조회 API 정의
- [ ] 분석 CRUD API 정의
- [ ] 통계 API (Top 분석 영화) 정의
- [ ] 추천 API (오늘의 추천 / 개인화) 정의
- [ ] 각 API별 상태 코드(200/201/401/403/404/500 등) 명시
- [ ] Request / Response JSON 예시 포함

---

### 1.3 DB 스키마(db.md)
- [ ] 핵심 테이블(users, movies, analyses) 정의됨
- [ ] 외부 영화 중복 기준(`external_movie_id UNIQUE`) 명시
- [ ] N:M 관계(movie_genres, analysis_tags) 정의됨
- [ ] 인덱스 전략(조회/페이징/통계) 명시
- [ ] 확장 테이블(통계/추천/이벤트) 고려됨
- [ ] soft delete / 시간 컬럼 정책 명시

---

### 1.4 정책(policy.md)
- [ ] 외부 영화 API 적재 정책 정의
- [ ] 최초 적재 / 재적재 기준 명시
- [ ] UPSERT 전략 명시
- [ ] 캐시 / 레이트리밋 / 장애 대응 정책 포함
- [ ] 커밋 컨벤션 정의
- [ ] 브랜치 전략 정의
- [ ] 이슈 관리 규칙 정의

---

## 8. 최종 목표 확인

> 이 프로젝트는 단순 CRUD 구현이 아니라  
> **“도메인 설계 + 확장 가능한 아키텍처 + 협업 가능한 문서화”**를 목표로 한다.

- [ ] 설계 의도를 말로 설명할 수 있다
- [ ] 왜 이런 구조를 선택했는지 설명할 수 있다
- [ ] 확장 시 변경 포인트를 알고 있다
